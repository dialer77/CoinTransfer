# 수취인 영문명 표기 가이드 (Travel Rule / 출금)

빗썸·OKX 등 거래소 출금 시 수취인 영문명(receiver_en_name 등) 입력 시 참고할 문서 요약입니다.

---

## 1. 거래소 공통 원칙

- **수취인 이름은 수취 거래소 계정에 등록된 이름과 동일**하게 입력해야 합니다.  
  (OKX, 바이낸스 등: “등록된 풀네임/법정 이름과 일치해야 검증 통과”)
- **여권/신분증 로마자 표기와 일치**하는 것이 안전합니다.  
  (OKX KYC·트래블룰: 여권 영문명과 불일치 시 출금/검증 실패 가능)

---

## 2. 대소문자

| 출처 | 내용 |
|------|------|
| **외교부 여권 로마자** | 국제적 표준으로 **대문자** 표기를 사용합니다. (예: HONG GILDONG) |
| **OKX / 일반 트래블룰** | “대문자만 사용”이라고 명시한 공식 문서는 없음. **해당 거래소에 등록된 표기 그대로** 입력하는 것이 원칙. |
| **IVMS-101** | 라틴 문자(A–Z, a–z), 숫자(0–9). 대소문자 강제 규정은 없고, **표준 필드명**은 lower camel case 등으로 정의. |

**실무 권장:**  
- 여권/거래소가 **대문자**로 되어 있으면 → **대문자**로 입력.  
- 거래소에 **First Name / Last Name** 형태로 저장돼 있으면 → 그 순서·띄어쓰기·대소문자를 **그대로** 맞추는 것이 가장 안전.

---

## 3. 띄어쓰기

| 출처 | 내용 |
|------|------|
| **일반** | 성과 이름 사이 **한 칸 띄어쓰기** (예: Hong Gildong, HONG GILDONG). |
| **빗썸 API (CCXT 이슈)** | `en_name` 등에 **공백이 있으면 JWT/인증 오류**가 난 사례가 있음. “Remove the white space from the en_name field” 권고. |
| **OKX** | “등록된 풀네임” 그대로 입력 → 등록 시 띄어쓰기가 있으면 동일하게 넣는 것이 맞음. |

**실무 권장:**  
- **빗썸** 출금: 공백 제거(연속 영문)로 시도해 볼 수 있음. (예: `HONG GILDONG` → `HONGGILDONG` 시도 시 인증 통과 사례 있음. 단, 수취 거래소가 “풀네임 띄어쓰기”로 검증하면 그쪽과 맞춰야 함.)  
- **OKX 등 수취측**: 해당 거래소 앱/웹에 표시되는 **정확한 영문명(띄어쓰기 포함)** 과 동일하게 입력.

---

## 4. 성·이름 순서

| 출처 | 내용 |
|------|------|
| **IVMS-101 (Global Travel Rule)** | 구조화 필드 기준: **primaryIdentifier = 성(Last Name)**, **secondaryIdentifier = 이름(First Name)**.  
  즉, “성 이름” 순서로 구분해 저장하는 구조. 단일 문자열(풀네임 한 칸)에 대한 “반드시 이 순서” 규정은 문서에 없음. |
| **한국 관습 / 여권** | 로마자 표기 시 보통 **성 + 이름** (예: HONG GILDONG). |
| **영문권 관습** | 서류/UI에선 **Given Name + Family Name** (이름 성) 순서도 많음. |
| **거래소** | **해당 거래소에 등록된 표기 순서**가 최종 기준. (바이낸스: First name / Last name 구분 시, 업비트 등으로 출금 시 “그 계정에 보이는 풀네임”과 일치 권장.) |

**실무 권장:**  
- **수취인 계정이 있는 거래소(OKX 등)** 에서 “이름이 어떻게 보이는지” 확인 후, **그 표기와 똑같이** 입력.  
- 한국인 여권 로마자면 보통 **성 이름** (예: HONG GILDONG)이 많음.

## 6. 요약 체크리스트

- [ ] **수취 거래소(OKX 등) 계정에 등록된 영문명**과 **완전 동일**하게 입력했는지 확인.
- [ ] 가능하면 **여권 로마자**와도 일치시키기 (대문자 사용 시 여권이 대문자면 대문자로).
- [ ] **빗썸** 출금 시 JWT/인증 오류 나면: 영문명에서 **공백 제거** 후 재시도 검토 (수취측 검증과 충돌하지 않을 경우).
- [ ] **성·이름 순서**는 “수취인 계정에 보이는 그대로”가 기준.
